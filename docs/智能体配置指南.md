# QuantMind 智能体（QuantBot/QwenPaw）配置指南

> 本文档描述 QuantBot 智能助手的初始化配置、能力范围和扩展方式。

---

## QuantBot 是什么

QuantBot 是 QuantMind 平台的智能助手，基于 **QwenPaw**（agentscope/qwenpaw）构建，可以通过自然语言对话完成数据同步、回测、训练、因子挖掘等操作。

## 架构关系

```
用户 ──→ QuantBot 界面 (Electron/Web)
              │
              ▼
       QwenPaw 容器 (:8088 → 外部 :8089)
              │
    ┌─────────┼──────────────────┐
    │         │                  │
    ▼         ▼                  ▼
  文件系统   Docker Socket    QuantMind API
  (代码/数据) (启动容器)     (http://quantmind:8000)
              │                  │
              ▼                  ▼
        量化引擎             量化引擎
```

### QwenPaw 容器已挂载的目录

| 容器路径 | 权限 | 说明 |
|----------|------|------|
| `/app/backend` | 只读 | 后端所有 Python 源码 |
| `/app/rd-agent` | 只读 | RD-Agent 完整源码 |
| `/app/scripts` | 只读 | 运维脚本 |
| `/app/strategy_templates` | 只读 | 11 个策略模板 |
| `/app/db` | 读写 | Qlib 数据 + parquet 源数据 |
| `/data` | 读写 | 运行时数据 |
| `/app/logs` | 读写 | 日志目录 |
| `/var/run/docker.sock` | 读写 | 可管理 Docker 容器 |

## 初始化配置

### Step 1: 确认 QwenPaw 运行正常

```bash
docker compose ps qwenpaw
# 状态应为 Up
```

访问 QwenPaw Web 界面: `http://<服务器IP>:8089`

### Step 2: 配置 API Key

QuantBot 需要 LLM API Key 才能进行自然语言对话。

**推荐配置** (DeepSeek):
```bash
# 在 .env 中
AI_IDE_LLM_API_KEY=sk-YOUR_DEEPSEEK_API_KEY
AI_IDE_LLM_BASE_URL=https://api.deepseek.com
AI_IDE_LLM_MODEL=deepseek-v4-pro
```

**备选方案** (Qwen/DashScope):
```bash
DASHSCOPE_API_KEY=sk-YOUR_DASHSCOPE_API_KEY
QWEN_API_KEY=sk-YOUR_QWEN_API_KEY
```

配置后重启服务：
```bash
docker compose up -d quantmind qwenpaw
```

### Step 3: 验证连接

```bash
# 检查 QuantBot 是否能访问 API
docker exec qwenpaw curl -s http://quantmind:8000/api/v1/health

# 检查数据库连接
docker exec quantmind python -c "
import asyncio
from backend.shared.db_manager_v2 import DatabaseManager
async def test():
    db = DatabaseManager()
    await db.initialize()
    print('DB OK')
asyncio.run(test())
"
```

### Step 4: 初始化 RD-Agent（如需要因子挖掘）

```bash
docker exec quantmind bash /app/scripts/setup/init.sh
```

验证：
```bash
docker exec quantmind python -c "
from rdagent.app.qlib_rd_loop.factor import *
print('RD-Agent ready')
"
```

## 扩展 QuantBot 能力

### 添加新的意图识别

编辑 `backend/services/engine/quantbot/intent_parser.py`，添加新的意图规则。

### 添加新的任务类型

编辑 `backend/services/engine/quantbot/rd_agent_launcher.py`，添加新的任务执行逻辑。

### 添加自定义 Skills（未来方向）

QuantBot 可以通过 Skills 机制扩展能力：

1. 在 `backend/services/engine/quantbot/skills/` 目录创建新的 skill 文件
2. 每个 skill 实现 `execute()` 方法
3. 在意图识别中添加对应的触发规则

示例 Skill 结构：
```python
# backend/services/engine/quantbot/skills/data_sync.py
async def execute(params: dict) -> dict:
    """执行数据同步任务"""
    # ...
    return {"success": True, "row_count": count}
```

## 故障排查

### QuantBot 无法对话

1. 检查 API Key 是否配置正确
2. 查看日志: `docker compose logs -f quantmind`
3. 验证 LLM 连接: `curl -s $AI_IDE_LLM_BASE_URL/chat/completions`

### QuantBot 无法执行操作

1. 检查服务间通信是否正常
2. 验证 `INTERNAL_CALL_SECRET` 配置
3. 检查 QwenPaw 容器挂载是否正确

### RD-Agent 因子挖掘不工作

1. 确认 `rdagent` 包已安装: `docker exec quantmind python -c "import rdagent"`
2. 确认 `/app/rd-agent` 目录存在
3. 查看日志: `docker compose logs -f quantmind`
