## 核心数据表：stock_daily_latest

这是系统最重要的数据表，包含每只股票每个交易日的所有数据。

### 表结构（89列）

| 分类 | 列名 | 类型 | 说明 |
|------|------|------|------|
| **主键** | trade_date, symbol | date, text | 交易日 + 股票代码 |
| **OHLCV** | open, high, low, close, volume, amount | float | 开高低收、成交量、成交额 |
| **涨跌停** | limit_up, limit_down, is_limit_up, is_limit_down | float, bool | 涨跌停价和标识 |
| **ST 标识** | is_st | boolean | 是否为 ST 股票 |
| **指数成分** | idx_hs300, idx_zz500, idx_zz1000, idx_margin | boolean | 沪深300/中证500/中证1000/融资融券 |
| **技术指标** | ma5, ma10, ma20, macd, rsi, kdj_*, boll_* | float | 常用技术指标 |
| **基本面** | pe, pb, total_mv, circ_mv | float | 市盈率、市净率、总市值、流通市值 |
| **其他** | turnover_rate, pct_change, amplitude 等 | float | 换手率、涨跌幅、振幅等 |
| **股票信息** | stock_name, industry | text | 股票名称、所属行业 |

### 数据来源

| 来源 | 文件 | 说明 |
|------|------|------|
| **本地源数据** | `db/custom/fundamental_aligned.parquet` | 720万行 × 88列，最全数据 |
| **远程数据库** | `SOURCE_DATABASE_URL` | 增量更新用 |
| **Qlib 数据** | `db/qlib_data/` | Qlib 格式数据，用于回测 |

## 初始化流程

### 场景 1: 首次部署（无数据）

#### Step 1: 安装 RD-Agent + 检查数据状态

```bash
docker exec quantmind bash /app/scripts/setup/init.sh
```

这会：
1. 检查 `/app/rd-agent` 是否挂载
2. 安装 `rdagent` Python 包
3. 检查 `stock_daily_latest` 是否有数据

#### Step 2: 全量同步数据（从 parquet）

```bash
# 同步最近 30 个交易日
docker exec quantmind python /app/scripts/data/maintenance/sync_stock_daily_full.py

# 同步更多天数（如 90 天）
docker exec quantmind SYNC_MAX_DAYS=90 python /app/scripts/data/maintenance/sync_stock_daily_full.py
```

验证结果：
```bash
docker exec quantmind-db psql -U quantmind -d quantmind -c "
SELECT trade_date, COUNT(*) as cnt
FROM stock_daily_latest
GROUP BY trade_date
ORDER BY trade_date DESC
LIMIT 5;
"
```

#### Step 3: 初始化数据库结构（如需要）

如果是全新的数据库（无表结构）：

```bash
# 初始化所有 93 张表
docker exec quantmind python -c "
import asyncio
from backend.shared.db_manager_v2 import DatabaseManager

async def init():
    db = DatabaseManager()
    await db.initialize()
    await db.ensure_all_tables()

asyncio.run(init())
"
```

#### Step 4: 下载 Qlib 数据

方式 A - 使用离线数据包：
```bash
# 下载后解压到 db/ 目录
# 详见 docs/数据包安装指南.md
```

方式 B - 从远程数据库下载：
```bash
docker exec quantmind python /app/scripts/daily_update.py --force
```

### 场景 2: 日常更新

#### 每日增量同步

```bash
# 从 parquet 同步最近交易日数据
docker exec quantmind python /app/scripts/data/maintenance/sync_stock_daily_full.py

# Qlib 数据增量更新
docker exec quantmind python /app/scripts/daily_update.py --incremental --source eltdx
```

### 场景 3: 数据排查

#### 检查表是否为空

```bash
docker exec quantmind-db psql -U quantmind -d quantmind -c "
SELECT 'stock_daily_latest' as table_name, COUNT(*) as row_count FROM stock_daily_latest
UNION ALL
SELECT 'users', COUNT(*) FROM users
UNION ALL
SELECT 'strategies', COUNT(*) FROM strategies
;"
```

#### 检查数据完整性

```bash
docker exec quantmind python -c "
import asyncio, os
from backend.shared.db_manager_v2 import DatabaseManager
from sqlalchemy import text

async def check():
    db = DatabaseManager()
    await db.initialize()
    async with db.get_session(read_only=True) as session:
        # 检查最新交易日
        r = await session.execute(text(\"\"\"
            SELECT trade_date, COUNT(*) as cnt
            FROM stock_daily_latest
            GROUP BY trade_date
            ORDER BY trade_date DESC LIMIT 3
        \"\"\"))
        for row in r.fetchall():
            print(f'{row[0]}: {row[1]} stocks')
        # 检查空列
        r = await session.execute(text(\"\"\"
            SELECT
                COUNT(*) as total,
                COUNT(is_st) as is_st_nn,
                SUM(CASE WHEN is_st=1 THEN 1 ELSE 0 END) as st_count,
                COUNT(idx_hs300) as hs300_nn,
                COUNT(idx_zz1000) as zz1000_nn,
                COUNT(idx_margin) as margin_nn
            FROM stock_daily_latest
            WHERE trade_date = (SELECT MAX(trade_date) FROM stock_daily_latest)
        \"\"\"))
        row = r.fetchone()
        print(f'Latest date: total={row[0]}, is_st_null={row[0]-row[1]}, st={row[2]}, hs300_null={row[0]-row[3]}, zz1000_null={row[0]-row[4]}, margin_null={row[0]-row[5]}')

asyncio.run(check())
"
```

## 故障排查

### 问题 1: stock_daily_latest 为空

**症状**: 智能策略筛选返回 0 条结果

**解决**:
```bash
docker exec quantmind python /app/scripts/data/maintenance/sync_stock_daily_full.py
```

### 问题 2: is_st/idx_hs300 等列为 NULL

**症状**: 排除 ST/沪深300 等筛选条件返回 0 条结果

**解决**: 运行全量同步，确保所有 86 列数据都已同步

### 问题 3: Qlib 回测报 "no data" 错误

**症状**: 回测失败，提示无数据

**解决**:
```bash
docker exec quantmind python /app/scripts/daily_update.py --force
```
